{{ m.icon("info", size=16, cls="text-3") }} How this works
1
A digital fingerprint of the file
When Cairn first sees a file, it calculates a hash (technically a “SHA-256”): a short, unique fingerprint computed from the file's exact contents. Change a single comma and the fingerprint changes completely, so it can prove a file hasn't been altered. The fingerprint can't be turned back into the file, so it reveals nothing about what's inside.
2
Recorded permanently on Bitcoin
That fingerprint is written into the Bitcoin blockchain, a public, worldwide ledger that is permanent and cannot be backdated or quietly edited. Cairn does this using OpenTimestamps, a free and open standard for timestamping. Only the fingerprint is published, never the file.
3
Anyone can check it, without trusting Cairn
Because the record is public, you, or an auditor, lawyer, or client, can confirm the file existed in this exact form by a given date, using the file plus its small proof file. The guarantee rests on Bitcoin and math, not on Cairn or any single company staying online.
